```

3. WALLET CONNECTION
   User → WalletConnect → TokenPocket → Polygon RPC → Wallet Added ✓

4. BALANCE FETCH
   ethers.js → Polygon RPC → Token Contract → Balance Returned ✓

5. DATA PERSISTENCE
   Portfolio → Firebase Realtime DB → Cloud Sync ✓

6. ACCOUNT RECOVERY
   Login → Firebase Auth → Portfolio Restored ✓

```
Firebase Realtime Database Structure:

users/
  {uid}/                             # User ID (auto-generated by Firebase)
    portfolio/
      ├── id: "abc123"              # Portfolio ID (same as uid)
      ├── wallets: [
      │   {
      │     address: "0x1234...5678",
      │     chainId: 137,            # Polygon
      │     connectedAt: 1707331200  # Unix timestamp
      │   },
      │   {
      │     address: "0xabcd...ef01",
      │     chainId: 137,
      │     connectedAt: 1707417600
      │   }
      │ ]
      ├── holdings: {
      │   "0x1234...5678": 100.5,    # CES balance for wallet 1
      │   "0xabcd...ef01": 50.0      # CES balance for wallet 2

      ├── buyPrice: {
      │   "0x1234...5678": 0.50,     # Average buy price for wallet 1
      │   "0xabcd...ef01": 0.75      # Average buy price for wallet 2
      │ }
      └── createdAt: 1707244800      # Account creation time

## 📊 Technical Details

### State Management
- **AuthContext** → Manages user, portfolio, current wallet
- **Component Props** → Pass down to Dashboard
- **Firebase Listeners** → Auto-update on remote changes

### Wallet Connection
- **WalletConnect v1** → Via `@walletconnect/web3-provider`
- **ethers.js** → For blockchain interactions
- **Polygon RPC** → Public RPC at `https://polygon-rpc.com/`

### Token Balance Fetching
```typescript
// Read-only call to CES token contract on Polygon
const balance = await contract.balanceOf(walletAddress);
// Returns balance in wei, converted to decimal
```

### Database Security
```json
{
  "rules": {
    "users": {
      "$uid": {
        ".read": "$uid === auth.uid",    // Can only read own data
        ".write": "$uid === auth.uid",   // Can only write own data
        "portfolio": {
          ".validate": "newData.hasChildren([...])" // Structure validation
        }
      }
    }
  }
}
```
